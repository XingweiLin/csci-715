Lighting In VR

Team Member: Gregory Ojiem, Xingwei Lin

Goal of the Project

The goal of this project was to create a lighting system for augmented reality objects that was based on the environmental lighting. Specifically making sure it lights AR objects well in environments with radically different lighting conditions. Ideally, it would include shadows, ambient lighting, and specular highlights, color information.

How It Was Built

Key steps in building this project included:

  • The lighting system was designed using a modular architecture within Unity, allowing for scalable and maintainable integration of various lighting components. We used HDR Light Estimation to accurately capture and represent the full spectrum of environmental lighting. This was achieved using device sensors and camera data to analyze the surrounding light conditions in real-time.
  • Utilized Unity's real-time shadow mapping techniques to cast realistic shadows from AR objects based on the detected environmental light sources.
  • Enabled adaptive color adjustments that respond to changes in ambient light, maintaining the visual integrity of AR objects under varying lighting conditions.

What to Do Next?

Future improvements for this project could include:

  • Enhance IDE Compatibility: Optimize the project for seamless integration across different Integrated Development Environments such as Android Studio and Xcode within Unity
  • Animation Optimization: Utilize more efficient animation algorithms and tools to enhance the smoothness and realism of animations while reducing resource consumption.
  • Cross-Device Optimization: Optimize performance for various hardware platforms, including different VR headsets and mobile devices, to ensure a consistent and high-quality user experience across all devices.
  • Enhance Light Simulation: Incorporate more sophisticated lighting simulation technologies, such as global illumination and real-time ray tracing, to achieve more realistic lighting effects.
